# Ruberra Matrix Discipline

## Trinity Matrix
- **Structure**: typed objects, continuity, index entries, recommendations.
- **Functionality**: transitions, transfer engine, export engine, resume engine, settings/connector/plugin mutations.
- **Visual**: chamber/profile surfaces rendering runtime truth.

## Chamber Matrix
- **School**: learning continuity, mastery reward paths, lab/creation transfer hooks.
- **Lab**: analysis continuity, evidence signals, school/creation transfer hooks.
- **Creation**: build continuity, export readiness, school/lab transfer hooks.

## Runtime Matrix Rules
- One object identity (`id`) across archive/profile/search.
- Recency merge strategy when seeded + runtime objects collide.
- Recommendations are generated from continuity state, not hard-coded UI assumptions.
