# RUBERRA — SINGLE-MAIN TASK FORCE DISCIPLINE

All pioneers work toward the same main truth, but not by editing main in uncontrolled parallel.

**Main is the sovereign trunk.**  
Execution must happen in controlled layered sequence.

---

## Order of Operation

| Phase | Pioneer | Responsibility |
|-------|---------|----------------|
| 1 | **Antigravity / Gemini** | Audits and defines the exact gap map |
| 2 | **Codex** | Implements the functional / runtime truth |
| 3 | **Cursor** | Refines and polishes the visible truth |
| 4 | **Copilot** | Assists with cleanup, consistency, and local finishing |
| 5 | **Final Audit** | Confirms no drift, no regression, no dead surfaces |

Each phase must be **audited and closed** before the next phase begins.

---

## Rules

- No pioneer may open a parallel product direction
- No pioneer may redesign the approved Ruberra root
- No pioneer may leave decorative controls alive
- No pioneer may overwrite another pioneer's domain without reason
- All changes must preserve the same Ruberra mother truth

---

## Domain Ownership

### Antigravity / Gemini
Strategic audit, taxonomies, content/system gap detection.  
Produces the gap map that governs what each subsequent pioneer builds.

### Codex
Runtime, continuity, object graph, profile, signals, search, settings, orchestration.  
Owns the functional and data layer. No visual redesign.

### Cursor
Shell, surfaces, chat/terminal polish, spacing, icons, motion, final visual closure.  
Owns the rendered layer. No logic rewrites.

### Copilot
Cleanup, refactor, small consistency fixes, edge cases.  
Operates after Cursor. No product direction changes.

## Prohibited Actions (All Pioneers)

- Do not open a new product direction
- Do not redesign the approved Ruberra root
- Do not leave dead decorative elements in the UI
- Do not overwrite another pioneer's domain without documented reason
- Do not merge unreviewed changes directly to main

---

## Definition of "Dead Surface"

A surface is dead if it:
- Cannot be clicked, expanded, filtered, or navigated
- Does not reveal, trigger, or connect to another meaningful object or state
- Exists purely as decoration with no functional destination

**Dead surfaces must be removed or wired before a phase is closed.**
