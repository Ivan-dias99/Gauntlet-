# Publishing Gauntlet — Edge first, then Chrome

This document is the operator playbook for shipping the Gauntlet
browser extension. We submit to the Microsoft Edge Add-ons store first
(faster review, friendlier reviewers), and then to the Chrome Web
Store.

## 0. Prerequisites

- [ ] Backend live at `https://ruberra-backend-jkpf-production.up.railway.app` and answering the four `/composer/*` routes.
- [ ] Privacy policy hosted on a public URL (see `PRIVACY_POLICY.md`).
      Recommended: a public GitHub Gist or a page on your site. The URL must be reachable without login.
- [ ] Microsoft Partner Center account (free): <https://partner.microsoft.com/dashboard/microsoftedge>
- [ ] Chrome Web Store Developer account (US$5 one-time): <https://chrome.google.com/webstore/devconsole>
- [ ] Replace the `[YOUR LEGAL NAME OR COMPANY]` and `[YOUR CONTACT EMAIL]` placeholders in `PRIVACY_POLICY.md`.

## 1. Build the production zip

From the repo root:

```bash
cd apps/browser-extension
npm install
npm run build      # writes .output/chrome-mv3/
npm run zip        # writes .output/<name>-<version>-chrome.zip
```

The `npm run zip` output is the file you upload to **both** stores —
Edge accepts standard Chromium MV3 zips with no transformation needed.

> Sanity check before uploading: unzip the artifact and confirm
> `manifest.json` has `"version": "1.0.0"` and the `host_permissions`
> list does **not** contain any `localhost` or `127.0.0.1` entries.

## 2. Required store assets

Both stores require the same set:

| Asset                 | Spec                                  | Status |
| --------------------- | ------------------------------------- | ------ |
| Store icon            | 128 × 128 PNG                         | ✅ `public/icon/128.png` |
| Screenshot ×1 minimum | 1280 × 800 or 640 × 400 PNG           | ⚠️ TODO — capture capsule on a real page |
| Promotional tile      | Edge: optional. Chrome: 440 × 280 PNG | ⚠️ TODO |
| Short description     | ≤ 132 characters                      | See suggested copy below |
| Long description      | ≥ 25 chars, recommended 1–4 paragraphs | See suggested copy below |
| Privacy policy URL    | public HTTPS URL                      | ⚠️ Host `PRIVACY_POLICY.md` somewhere public |

### Suggested short description

> Press Ctrl+Shift+Space anywhere on the web to summon Gauntlet — a cursor capsule that reads the page and helps you act on it.

### Suggested long description

> Gauntlet is a cursor-first assistant that lives on top of every web
> page. Press Ctrl+Shift+Space (or click the toolbar icon) and a small
> capsule appears next to your cursor. Type what you want to do —
> rewrite a paragraph, summarize the article, draft a reply, or plan
> a sequence of clicks on the page — and Gauntlet uses the page
> context plus your selection to produce a precise answer or a list
> of DOM actions you can preview before applying.
>
> Gauntlet does not run in the background, does not track your
> browsing, and does not display ads. The capsule is dormant until
> you summon it. When you do, the page URL, title, your selection,
> and a simplified DOM are sent to the Gauntlet backend to generate
> the response. Full details: see the privacy policy linked on this
> page.

## 3. Microsoft Edge Add-ons (first)

1. Go to <https://partner.microsoft.com/dashboard/microsoftedge/public/extensions>.
2. **Create new extension** → upload the zip from step 1.
3. **Availability**: choose markets (recommended: all).
4. **Properties**:
   - Category: `Productivity`
   - Privacy policy URL: paste your public URL.
   - Mature content: No.
   - Permissions justification: paste section 2 of `PRIVACY_POLICY.md`
     (the permission table) into the "Why does your extension require
     each permission?" textbox.
5. **Store listing**: short + long description from §2; upload icon
   and at least one 1280×800 screenshot.
6. **Submission notes** to reviewers (textbox at the end):
   ```
   Test instructions:
   1. Install the extension.
   2. Visit any HTTPS page (e.g. https://en.wikipedia.org/wiki/Electron).
   3. Press Ctrl+Shift+Space. A capsule appears near the cursor.
   4. Type "summarize this page" and press Enter.
   5. The capsule shows a response generated by our backend.
   The backend is at ruberra-backend-jkpf-production.up.railway.app
   and is operated by us.
   ```
7. **Submit for certification**. Edge review typically takes 1–3
   business days.

## 4. Chrome Web Store (after Edge approval)

1. Go to <https://chrome.google.com/webstore/devconsole>.
2. **New item** → upload the **same** zip from step 1.
3. Fill the **Store listing** with the same copy and assets used on
   Edge.
4. **Privacy practices** tab — this is where Chrome is strict:
   - **Single purpose**: "Provide an on-page AI capsule that reads the
     active tab on user invocation and returns assistant responses or
     a DOM-action plan."
   - For each permission (`activeTab`, `tabs`, `scripting`, `storage`,
     `<all_urls>`), paste the matching row from the permission table
     in `PRIVACY_POLICY.md` §2.
   - **Are you using remote code?** No (all logic is bundled).
   - **Data handling**: declare *Website content*, *User activity*,
     and *Personally identifiable information* (because URLs may
     contain it) as collected; check "Encrypted in transit" and
     "Not sold to third parties".
   - Privacy policy URL: paste the same public URL.
5. **Distribution** → Public (or Unlisted if you want a soft launch).
6. **Submit for review**. Chrome review for `<all_urls>` extensions
   typically takes 5–14 days, sometimes longer.

## 5. After approval

- Tag the release in git: `git tag composer-v1.0.0 && git push --tags`.
- Bump `version` in `package.json` for the next iteration (e.g.
  `1.0.1`) so the next zip is accepted by both stores (they reject
  re-uploads with the same version).
- Watch the backend logs for the first 24 h after listing goes live.
