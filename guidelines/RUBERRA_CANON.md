# Ruberra Origin Canon — Inspirations, MVP, and First Truth

## Verdade Central
O trabalho sério não deveria acontecer em pedaços. Ruberra não é SaaS, dashboard ou wrapper. É um **organismo operacional soberano**, construído sob a Lei de Consequência: tudo nasce do contexto, pertence a uma missão, liga-se à memória e produz consequência.

## O Que Ruberra É (Non-Negotiables)
- **Sovereign**: Roteável, livre de vendor lock-in, memória própria.
- **Mission-bound**: Trabalho focado em missões, não tarefas mortas.
- **Memory-bearing**: Memória como substrato ativo. Não esquece.
- **Chamber-native**: Domínios vivos persistentes (Lab, School, Creation, Profile).
- **Consequence-driven**: Muda o estado do sistema com a execução.
- **Anti-fragmentation**: Reduz exits estruturais, centraliza o fluxo.

## Drifts Proibidos (Kill List)
- Painéis admin genéricos, SaaS card walls, dashboards de KPI.
- Chat-first layouts, blocos de chat descartáveis.
- Estética sci-fi, glow-heavy, gamer control room ou neon.
- Branco estéril corporativo ou dark mode fintech.

## Visual DNA Lock
Ruberra é **White Premium** e **Dark Premium**.
- **Inspiração visual**: Linear (shell discipline), Cursor (quiet confidence), Raycast (command precision), Apple/Vision Pro (spatial restraint).
- **Tipografia**: Sérias, sem decorações vazias. JetBrains Mono para system/code, Inter para UI. Board radius de 2px ("Mineral Shell").

## Superfícies Essenciais (Trindade Operacional)
1. **Home / Dashboard**: Superfície de re-entry operacional. Pressão, memória, blocos de continuação.
2. **Object Detail / Entity Truth**: Estado durável, histórico de atividades, dependências no Mesh.
3. **Command Surface**: Terminal-grade execution. Action throat of the organism. Input focado em consequência.
4. **Campaign / Thread View**: Execução viva (Status, Timeline, Closure Pressure).

## 4 Portas do Organismo
- **Lab**: Terminal de inteligência aplicada (análise, hipótese).
- **School**: Motor soberano de continuidade (deep study, future role tracks).
- **Creation**: Superfície de saída e manifestação de assets.
- **Profile / Operator**: Cockpit estrutural.
