# Tracked eval results — promoted by CI

This directory is **committed** to git, unlike `ruberra-backend/data/evals/`
which is gitignored and only holds local-run output.

The GitHub Actions `evals` workflow writes `latest.json`, `history.json`,
and `latest-baseline.json` into this directory after every successful
run and pushes the commit to `main`. The FastAPI backend reads from
here as a fallback so a freshly-deployed production instance surfaces
the most recent nightly eval in the Memory chamber even before any
local runs have happened.

Do not edit these files by hand — they're regenerated by CI.
