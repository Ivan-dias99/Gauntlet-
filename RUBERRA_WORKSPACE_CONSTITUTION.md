## SECTION 9 — EXECUTION DISCIPLINE (CASCADE LAW)

1. Inspect latest relevant state only
2. Inspect what was completed in the owned frontier
3. Inspect what remained open
4. Preserve what is real — real work, real code, real consequence
5. Remove what is false — duplicates, dead surfaces, weak patterns
6. Implement only the true frontier — the next real thing
7. Verify — build passes, runtime passes, consequence is real
8. Repair if partial — partial completion is not completion
9. Advance — only then open the next frontier

**Violations:**
- Opening a new task while prior residue is open
- Implementing decorative surfaces without consequence
- Merging work that fails build or runtime
- Adding features outside the current owned frontier
- Drifting toward the CANNOT list
