# SESSION BOOTSTRAP — Genesis Core

Run this preflight before any material work block.

## Read Order

1. `CURRENT_RUNTIME_AUTHORITY.md`
2. `ops/CANONICAL_TRUTH.md`
3. `ops/LIVE_STATE.md`
4. `ops/MILESTONE_TRACKER.md`
5. `ops/HANDOFF_LEDGER.md` (latest relevant entries)
6. `ops/SALVAGE_MAP.md` (only when importing from external or legacy repos)

## Identity Block

Declare this at the top of any serious execution session:

```text
WAVE:
ROLE:
SURFACE:
ACTIVE ORGANISM: src/ruberra/
TASK:
OBJECTIVE:
ACCEPTANCE CONDITION:
```

## Hard Gates

- If the work does not strengthen `src/ruberra/`, stop.
- If the work re-legitimizes `src/app/`, stop.
- If the work creates a second product center, stop.
- If the work cannot name its acceptance condition, stop.
- If the work is import/salvage, check `ops/SALVAGE_MAP.md` first.

## Surface Types

Use one of these in the identity block:

- `product`
- `ops`
- `visual`
- `architecture`
- `audit`

## Rule

Bootstrap exists to reduce re-framing cost.
It is not ceremony.
If it does not sharpen the next move, compress it.
