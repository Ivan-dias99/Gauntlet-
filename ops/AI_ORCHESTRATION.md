# AI ORCHESTRATION — RUBERRA GENESIS ERA

## Purpose

AI exists inside Ruberra as force multiplication for product truth, not as decorative plurality.

## Command Law

One active command center:
- Ivan direction
- Builder execution pressure
- Audit merge truth
- Architect structural truth
- Visual surface truth

## External AI Routing Principle

Do not multiply agents for theater.
Only activate an agent if it provides a distinct gain in one of these:

1. architecture
2. build execution
3. visual correction
4. audit pressure
5. repo truth compression

## Selection Standard

Every external AI must be judged by:

- reliability under repo work
- ability to follow canon
- handoff quality
- non-generic output
- usefulness to active wave

## Preliminary Pioneer Buckets

### Sovereign / closed-weight roles
- strategic command intelligence
- product truth compression
- architectural judgment
- final audit pressure

### Open-weight roles
- implementation acceleration
- codebase navigation
- patch generation
- repo-local search and execution help

## Ruberra Rule

AI orchestration must remain subordinate to product truth.
No AI layer may become a second center of gravity.
