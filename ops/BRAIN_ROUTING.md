# BRAIN ROUTING

This file defines when the operational layer must consult the Ivan Brain.

## Brain routing triggers

Consult the brain before:
- changing product identity
- changing chamber semantics
- changing runtime authority
- changing merge discipline
- changing security posture
- changing memory/canon rules
- deciding between multiple plausible founder-intent paths
- absorbing external pain into Ruberra scope

## Minimum route

1. `brain/00_index/QUERY_INDEX.md`
2. relevant Ivan core node(s)
3. relevant Ruberra core node(s)
4. relevant protocol/security node(s)
5. `brain/04_execution/DECISIONS_LEDGER.md`

## Routing law

If the brain already contains the answer, do not improvise.
If the brain does not contain the answer, act carefully and write back only if new truth was materially discovered.

## Bridge rule

`ops/NEXT_ACTION.md`, merge cleanup, and any founder-intent dispatch should assume brain consultation is part of the decision path, not optional decoration.
