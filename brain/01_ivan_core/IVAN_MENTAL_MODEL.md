# IVAN MENTAL MODEL

## Primary thinking shape

Ivan thinks in top-down layered systems.
He prefers:
- organism before interface
- principle before feature
- command before convenience
- consequence before decoration
- integration before fragmentation

## Invention heuristics

When facing a problem, Ivan tends to:
- look for an overlooked substrate
- use what is abundant before what is scarce
- reuse existing infrastructure before building redundant new infrastructure
- optimize the observer / control layer, not only the observed layer
- search for a cleaner alternate route instead of polishing the obvious one

## Quality bar

Ivan's instinctive bar is not "acceptable".
It is podium-level inevitability.

That means output should feel:
- stronger than category average
- less generic than market norms
- more coherent than stacked-tool improvisation
- more structurally mature than surface polish alone

## Failure signatures

Ivan rejects output that feels like:
- SaaS template logic
- chat wrapper logic
- dashboard symmetry without hierarchy
- verbose explanation without delta
- partial completion disguised as done
- multiple competing truths living at once
