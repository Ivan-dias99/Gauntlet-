# RUBERRA SHELL LAWS

The shell is not generic navigation.
The shell is the ritualized operational frame of the organism.

## Shell must do
- bind the user into a live state
- reveal continuity
- expose persistent system condition
- frame chamber gravity without flattening it
- preserve consequence signals across movement

## Shell must not become
- dashboard toolbar
- hero surface
- metrics wallpaper
- equal-weight tab system
- decorative premium theater

## Hierarchy law
The shell should read in this order:
1. ritual entry / return recognition
2. system spine / atmospheric condition
3. active chamber gravity
4. truth and memory rails
5. secondary controls

## Visual law
The shell should feel:
- sovereign
- calm but dense
- premium by judgment
- atmospheric, not noisy
- structurally clear

## Ruberra note
If the shell starts feeling like a generic app frame, drift has already begun.
