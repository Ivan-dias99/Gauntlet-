# TAG SYSTEM

Use tags to make the Ivan Brain queryable without flattening it into database theater.

## Core tags
- `identity`
- `canon`
- `law`
- `pain`
- `security`
- `protocol`
- `merge`
- `wave`
- `rejection`
- `tension`
- `memory`
- `agent`
- `future`

## Usage law
Tags should clarify query paths.
They should not replace structural folders and named documents.

## Suggested pattern
A file may implicitly belong to several tags, but should clearly center one dominant function.
Example:
- `RUBERRA_CREATION_LAWS` → `canon`, `law`, `creation`
- `MERGE_LEDGER` → `merge`, `execution`
- `ANTI_DRIFT_PROTOCOL` → `protocol`, `law`, `identity`
