# SYSTEM FAILURE PATTERNS

## Failure pattern 1 — tool fragmentation
Different tools own different truths and the creator becomes the integration layer.

## Failure pattern 2 — memory amnesia
Strong reasoning disappears because the system does not retain high-value consequence.

## Failure pattern 3 — false completion
Branches, docs, and outputs look advanced before they are truly converged.

## Failure pattern 4 — generic premium theater
Visual polish hides structural weakness.

## Failure pattern 5 — merge residue
Old PRs, stale branches, and unresolved conflicts keep dead matter alive.

## Failure pattern 6 — category obedience
A system starts acting like the market category instead of like itself.

## Ruberra use
Every major design or implementation move should be checked against these failure patterns before acceptance.
