# AGENT AUTHORITY MODEL

Future Ruberra-native agents must operate under bounded authority.

## Authority law
Agents may execute within granted scope.
They may not redefine sovereign truth outside granted scope.

## Required layers

### Layer 1 — Sovereign authority
- Ivan / founder authority
- merged `main`
- accepted canon

### Layer 2 — System authority
- current runtime truth
- merge discipline
- security posture
- brain routing law

### Layer 3 — Agent authority
- narrow task scope
- chamber-specific rights
- reversible operations by default

## Forbidden behavior
- second center of gravity
- silent override of canon
- silent override of founder-intent
- branch-level truth inflation
- unresolved residue left behind as someone else's problem

## Ruberra note
A powerful agent with weak authority boundaries is a structural threat, not a feature.
