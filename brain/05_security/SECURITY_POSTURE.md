# SECURITY POSTURE — IVAN BRAIN / RUBERRA

This is not narrow auth-only security.
This is structural security.

## Security domains

### Authority security
Protects against competing truths.

### Mutation security
Protects canon, memory, and direction from casual corruption.

### Execution security
Protects the system from ambiguous or silent consequence.

### Agent security
Protects against lane drift, second authority, and unresolved residue.

### Recovery security
Protects continuity under conflict, stale branches, or bad merges.

## Eternal Nexus inheritance

The security posture inherited from the Eternal Nexus discipline requires:
- explicit trust boundaries
- explicit permissions
- no silent mutation
- no fake completion
- no uncontrolled merge residue
- recoverability after conflict
- sovereign authority preserved under pressure

## Ruberra law

Security is not separate from product truth.
If authority is split, merge discipline fails, or memory can drift silently, the system is insecure even if auth is fine.
