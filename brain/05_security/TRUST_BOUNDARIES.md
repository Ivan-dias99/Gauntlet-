# TRUST BOUNDARIES

## Boundary 1 — sovereign main
Merged `main` outranks preview, branch, and open PR.

## Boundary 2 — living organism
`src/ruberra/` outranks legacy quarantine.

## Boundary 3 — canon
Canon outranks ad-hoc branch preference.

## Boundary 4 — brain
The Ivan Brain outranks model improvisation on high-consequence direction.

## Boundary 5 — lane ownership
A lane may act within its scope.
A lane may not silently redefine repo truth outside that scope.

## Boundary 6 — destructive change
Destructive or irreversible action requires explicit justification and confirmation path.

## Boundary 7 — unresolved PR residue
Open/conflicted/stale PR state must never be treated as harmless background noise.

## Result
Any action that crosses these boundaries without explicit right should be treated as drift, breach, or invalid execution.
