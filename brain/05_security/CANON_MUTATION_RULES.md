# CANON MUTATION RULES

## Law
Canon must not mutate casually.

## Valid mutation triggers
- reality changed materially
- stronger truth displaced weaker truth
- contradiction was proven and resolved
- founder-authorized direction matured

## Invalid mutation triggers
- branch preference
- preview excitement
- contributor convenience
- trend imitation
- aesthetic preference alone

## Required conditions
Canon mutation should be:
- explicit
- attributable
- reasoned
- recoverable in lineage

## Security note
Unclear canon mutation is a structural security failure, not merely a documentation issue.
