# RECOVERY PROTOCOL

## Law
A sovereign system must recover cleanly after conflict, drift, or bad merge state.

## Recovery triggers
- stale PR residue
- bad branch divergence
- authority confusion
- canon contradiction
- memory corruption risk
- preview mistaken for product truth

## Recovery sequence
1. identify sovereign truth (`main`, living organism, accepted canon)
2. identify residue
3. harvest what is still valid
4. close or archive what is no longer valid
5. restore clean narrative and clean runtime authority

## Ruberra note
Recovery is not just rollback.
Recovery is restoration of clean system truth.
