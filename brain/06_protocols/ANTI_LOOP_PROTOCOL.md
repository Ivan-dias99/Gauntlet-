# ANTI-LOOP PROTOCOL

## Law

A system is looping when it keeps moving matter without closing consequence.

## Loop signatures
- PR open after substance already converged
- task reported done without clean repo end-state
- same problem reframed repeatedly without harder decision
- branch work living too long without merge, harvest, or closure
- repeated summaries replacing actual closure

## Required behavior
- converge or close
- harvest or kill
- merge or reject
- never let floating residue masquerade as progress

## Pioneer rule
If your work leaves unresolved residue behind you, the task is not complete.
