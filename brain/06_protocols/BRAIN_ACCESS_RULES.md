# BRAIN ACCESS RULES

## Law

The brain exists to be consulted before high-consequence action.

## Pioneers must read the brain before:
- redefining product direction
- changing authority or merge discipline
- proposing structural shifts
- touching security posture
- inventing new chamber semantics
- acting under ambiguity on founder-intent questions

## Minimum read path
1. `brain/00_index/QUERY_INDEX.md`
2. relevant core node(s)
3. relevant protocol(s)
4. relevant execution/security node(s)

## After action
A pioneer should write back to brain only when there is real new law, realization, or decision worth preserving.

## Forbidden use
- do not use the brain as decorative note storage
- do not rewrite the brain casually
- do not invent founder intent when the brain already defines it
