## PR #221 — `claude/wave-b-intent-switch-guard`
**Round informado:** `anchor diacritics + negação`

### Parecer
**Status sugerido: REQUEST CHANGES (1 ajuste objetivo).**

### Razão
Não consegui localizar evidência direta no estado atual de uma suíte de testes dedicada ao guard de intent-switch com cenários de:
- diacríticos (ex.: “não”, “ação”, “conexão”),
- negação contextual (“não trocar”, “não mude”, etc.),
- falsos positivos com termos âncora parciais.

Sem esse teste, o comportamento pode parecer correto em happy path mas degradar em edge cases linguísticos.

### Comentário que eu deixaria no PR
- _Blocking comment:_ adicionar testes de tabela (table-driven) com pares `input -> expected_intent` cobrindo diacríticos e negação em PT-BR, incluindo casos ambíguos e double-negation.
